# Venue card

## Overview

Venue card is the main card component used in the customer app to display partner venues. Each card shows the venue photo, name, category, and rating. When a venue is a partner, the card also displays a [Badge Partner](/components/ds-customer/indicators-and-states/badge-partner/index) badge overlaid on the image to communicate booking capability (bookable or request only).

Venue card is a display-only component. It is not interactive at the component level — tap/press behavior is handled at the container level.

## Anatomy

Venue card is composed of five parts:

| Part | Required | Description |
|---|---|---|
| **Image** | Yes | Full-width venue photo at the top of the card. Displays a fallback if no image is available. |
| **Badge Partner** | Conditional | Overlaid on the image, top-left area. Only shown when `Partner=True`. Communicates booking type (Bookable or Request only). |
| **Venue name** | Yes | Bold text below the image. Primary identifier of the venue. |
| **Category** | Yes | Subtitle text below the venue name. Describes the type of venue (e.g. "Beach Club & Restaurant"). |
| **Rating row** | Yes | Star icon, numeric rating, and review count. Gives social proof at a glance. |

## Variants

Venue card has one prop: `Partner`.

| Variant | Prop | Description |
|---|---|---|
| **Partner** | `Partner=True` | Shows the Badge Partner badge overlaid on the venue image. Use for partner venues. |
| **Standard** | `Partner=False` | No badge. Use for non-partner venues or when booking capability is not applicable. |

> The `Partner` prop controls the visibility of the Badge Partner badge only. All other parts of the card remain identical between variants.

## Usage guidelines

- Use Venue card to display partner venues in lists, grids, or carousels in the customer app.
- Set `Partner=True` when the venue is a partner and has a booking capability defined. The Badge Partner badge will appear automatically.
- Set `Partner=False` for non-partner venues or when booking capability data is not available.
- Do not display both `Bookable` and `Request only` badges on the same card simultaneously — the badge type is controlled by the Badge Partner component's `type` prop.
- Keep venue name and category concise — the card layout does not accommodate long wrapping text.
- Always provide a venue image. If no image is available, the `ImageWithFallback` layer handles the fallback state.
- Do not repurpose Venue card for non-venue content (e.g. events, products) — use the appropriate card component instead.

## Accessibility

- Provide a descriptive `alt` text for the venue image that includes the venue name (e.g. `alt="Blue Marlin – Beach Club & Restaurant"`).
- The Badge Partner badge icon is decorative — ensure `aria-hidden="true"` is applied to it. The badge label text ("Bookable" / "Request only") carries the meaning.
- The rating row should be announced meaningfully — consider `aria-label="4.8 stars, 342 reviews"` on the rating element rather than reading the star icon as a character.
- Do not rely on the badge color alone to convey booking type — the label text is always required.

## Do / Don't

**Do:**
- Use `Partner=True` only for venues with a confirmed partner status and booking capability.
- Always show venue name, category, and rating — they are all required parts.
- Use the Badge Partner badge to communicate booking type clearly.

**Don't:**
- Don't use Venue card for non-partner venues that don't have the required metadata (name, category, rating).
- Don't modify the fixed labels of the Badge Partner badge ("Bookable" / "Request only").
- Don't place Venue card outside of a proper list, grid, or carousel container.
- Don't truncate the rating row — always show the full rating and review count.
