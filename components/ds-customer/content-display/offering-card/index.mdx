# Offering Card

## Overview

Offering Card displays a single bookable offering within a venue's listing. It communicates the offering's title, description, booking method, price, guest capacity, and current availability at a glance. It is a display-only component — taps are handled by the parent list item.

Used on venue detail screens and offering selection flows in the Customer App.

## Anatomy

Offering Card is composed of three areas:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | Card surface. Background: `surface/neutral/raised`. Border radius: 24px. Padding: 16px top/bottom, 16px left, 12px right. |
| **Image** | Optional | 128×128px venue photo. In `sold-out` state, a 40% white overlay dims the image. |
| **Content column** | Yes | Flex column with three sub-rows: `col.top` (title + badge + description), `col.pills` (0–2 badge pills), `col.bottom` (price + guest count). |

- The image is optional. When absent, the content column fills the full card width.
- The content column always shows all three sub-rows — layout does not collapse.

## Variants

Offering Card has two independent variant axes:

### `booking` — booking method and payment model

| Value | Booking flow | Payment model | Badge (slot 1) |
|---|---|---|---|
| `free-instant` | Confirmed immediately | Free | Cyan "Instant" |
| `request-quote` | Venue must accept | Price quoted by venue | Pink "Limited" |
| `request-min-spend` | Venue must accept | Minimum spend required | Pink "Limited" |
| `instant-fixed` | Confirmed immediately | Fixed price | Cyan "Instant" |
| `all-day-instant` | Confirmed immediately | Fixed price, full-day | Cyan "Instant" + Success "All day" |

### `state` — current availability

| Value | Meaning | Visual effect |
|---|---|---|
| `available` | Fully bookable | No availability overlay |
| `limited` | Few spots remaining | Adds Warning "2 left" badge in slot 2 |
| `sold-out` | No availability | Image dimmed, price replaced with "SOLD OUT", badge replaced with gray "Sold out" |

**Total combinations:** 5 × 3 = 15. Use `booking` to match the actual booking flow, and `state` to reflect live availability data.

> Note: `all-day-instant` in `limited` state shows "2 left" in slot 2 instead of "All day".

## Usage guidelines

- Use `booking` to match the **actual booking confirmation method** of the offering — not a desired label.
- Use `state` to reflect **live availability data** — only show `limited` when an actual count is available, only show `sold-out` when confirmed.
- Show a maximum of one Offering Card per offering. Do not show the same offering in multiple states simultaneously.
- Keep the title to 1–4 words. Keep the description to 1–2 lines (40–60 characters).
- Do not use `request-*` booking values for offerings that confirm instantly.
- Do not show a price in `request-quote` — always use the two-line "Quote / required" display.
- Do not modify badge colors — use the exact badge types defined per booking and state combination.

## Accessibility

- The card is a display-only component. Interactivity belongs on the parent list item — ensure the parent has an `accessibilityRole` and an `accessibilityLabel` that includes the offering title.
- Do not rely on badge color alone to communicate booking method or availability — the badge label text must carry the meaning independently.
- The dimmed image in `sold-out` is decorative — ensure the "SOLD OUT" text in the price area is read by screen readers.

## Do / Don't

**Do:**
- Match `booking` to the actual booking flow the venue uses.
- Use `state=limited` only when you have a real count to display.
- Keep title and description concise — the card is a preview, not a detail page.
- Ensure the parent list item handles the tap interaction with a proper accessibility label.

**Don't:**
- Don't set `booking=request-quote` for an offering that books instantly.
- Don't show a numeric price in `request-quote` — show "Quote required" instead.
- Don't override badge types manually — use the defined combination per booking and state.
- Don't hide the image in `sold-out` — use the 40% overlay dimming instead.
- Don't use this card to display non-bookable content (information-only items).
