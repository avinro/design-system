# Header Home

## Overview

Header Home is the top navigation component used exclusively on the Home screen of the customer app. It communicates the user's current plan at a glance and provides quick access to the Chats page and the Account page.

The component is composed of three parts: a chat icon button on the left that navigates to the Chats page, a central event pill that reflects the current state, and a user avatar on the right that navigates to the Account page. The event pill updates in real time based on the user's active bookings or ride status.

Header Home is not used on any screen other than Home.

## Anatomy

Header Home has three fixed parts:

| Part | Required | Description |
|---|---|---|
| **Chat button** | Yes | Secondary icon-only button on the left (44px). Navigates to the Chats page. |
| **Event pill** | Yes | Central pill element (44px height). Displays the current plan state with an optional icon, title, and subtitle. Tappable — navigates to the relevant plan or ride detail. |
| **Avatar** | Yes | Circular user avatar on the right (40px, size MD). Shows the user's profile photo. Tapping navigates to the Account page. |

The event pill fills the available width between the chat button and avatar. Its background and icon change depending on the active state.

## States

Header Home has a single `State` prop with four values. Only one state is shown at a time, determined by real-time data.

| State | Pill icon | Pill background | Title | Subtitle |
|---|---|---|---|---|
| **Add event** | Calendar | Translucent | "No plans for today" | "Add a event →" |
| **Pick up** | Calendar | Translucent | Event / venue name | "Pick up at [time]" |
| **Reservation** | Calendar | Translucent | Event / venue name | "Reserve at [time]" |
| **Ride** | Car | Solid blue | "On the way" | Destination address |

> The Ride state is the only state that uses a solid blue pill background (`app-header/ride-pill/bg`). All other states use the translucent background (`app-header/event-pill/bg`).

## Usage guidelines

- Use Header Home only on the Home screen. It must not appear on other screens.
- Display the correct state based on real-time data: no plan → `Add event`, event with pickup → `Pick up`, event with reservation → `Reservation`, active ride → `Ride`.
- The Ride state must always use the blue pill background. Do not apply it to other states.
- Keep pill text short — title and subtitle only. Do not add extra lines or truncate content.
- The chat button always navigates to the Chats page. Do not repurpose it for other actions.
- The avatar always navigates to the Account page. Do not repurpose it for other actions.
- The avatar always shows the current user's profile photo. Always provide a fallback.
- Do not reorder or hide the three parts (chat button, pill, avatar).

## Accessibility

- **Chat button:** provide `aria-label="Open chats"` (or equivalent localized string).
- **Event pill:** when tappable, provide an `aria-label` that describes the full state context, e.g. `aria-label="Dinner at Amante – Pick up at 5pm"` or `aria-label="No plans for today. Add an event."`.
- **Avatar:** provide `aria-label="Go to account"` on the tap target and `alt` text with the user's name on the image (e.g. `alt="Profile photo"`).
- Do not rely on icon type or pill color alone to convey state — the text labels are always required.
- The calendar and car icons inside the pill are decorative — apply `aria-hidden="true"` to them.

## Do / Don't

**Do:**
- Show only one state at a time based on real-time data.
- Use the blue pill background exclusively for the Ride state.
- Keep all three parts visible at all times.
- Use the fixed label text: "No plans for today" and "On the way" are not user-configurable.

**Don't:**
- Don't reuse Header Home outside the Home screen.
- Don't apply the blue pill background to any state other than Ride.
- Don't add extra icons, badges, or content to the header.
- Don't change the "Add a event →" CTA label or remove the arrow.
