# Select Boolean

## Overview

Select Boolean is a binary choice input that presents a question label and two mutually exclusive pill-shaped option buttons laid out side by side. It is used in forms where the user must answer a yes/no or equivalent two-option question. Exactly one option is always active ‚Äî selecting one automatically deselects the other.

It uses the same button styling tokens as the core Button component: the selected option uses the Default (filled) button style, and the unselected option uses the Secondary (transparent) button style.

## Anatomy

The component has two parts stacked vertically:

1. **Question Label** ‚Äî a short text question above the option group. Uses the same label style as other input components in this library.
2. **Option Group** ‚Äî a horizontal row of two equal-width pill buttons:
   - **Option A** (left) ‚Äî the first choice
   - **Option B** (right) ‚Äî the second choice

Each option button displays:
- **Label text** ‚Äî the option name (e.g. "Yes", "No")
- **Fill** ‚Äî dark filled when selected, transparent when unselected

## States

The selected state alternates between the two options. There is no unselected/empty state ‚Äî one option is always active.

| State | Option A | Option B |
|-------|----------|----------|
| **Option A selected** | Filled ‚Äî `--button/default/bg-default` | Transparent ‚Äî `--button/secondary/bg-default` |
| **Option B selected** | Transparent ‚Äî `--button/secondary/bg-default` | Filled ‚Äî `--button/default/bg-default` |

## Usage guidelines

- Use Select Boolean when the user must answer a binary question ‚Äî two options, one required answer.
- Always include the question label. It must clearly state what is being decided.
- Do not use Select Boolean for more than two options ‚Äî use a segmented control or radio group instead.
- Do not use Select Boolean for open-ended choices where neither option should be pre-selected. If a no-default state is needed, use a different input pattern.
- The two options must be mutually exclusive and collectively exhaustive ‚Äî they must cover all valid answers to the question.
- Keep option labels short: one or two words at most. The question label carries the context.
- Use sentence-style capitalisation for the question label. Use title case for option labels.

## Accessibility

- The option group should be implemented as a `radiogroup` with each button as a `radio` input, or as a pair of toggle buttons with `aria-pressed`.
- The question label must be associated with the group via `aria-labelledby` or `fieldset`/`legend`.
- The currently selected option must be communicated to assistive technology (e.g. `aria-checked="true"` or `aria-pressed="true"`).
- Both options must be reachable and operable via keyboard.

## Do / Don't

**Do:**
- Write the question label as a direct question: "Do you have accommodation?" not "Accommodation"
- Keep option labels to 1‚Äì2 words each
- Always pre-select the most common or safe default

**Don't:**
- Use Select Boolean for more than two options
- Use generic option labels like "Option 1 / Option 2" ‚Äî always use meaningful labels
- Stack multiple Select Boolean components without clear visual separation between questions
