# Chat Input

## Overview

The Chat Input is a single-line text entry field used in chat and messaging screens. It is composed of a pill-shaped input field and a circular send button. The send button is visually passive when the field is empty and changes to a filled, prominent style once the user begins typing — providing a clear affordance for sending.

It is used in both the Customer App and Driver App for in-ride messaging between riders and drivers.

## Anatomy

The Chat Input has three parts:

- **Wrapper** — the outer container. Carries the surface background color and the layout spacing between the input pill and the send button.
- **Input pill** — the pill-shaped text field. Displays placeholder text when empty and typed text when active. The border color changes between states.
- **Send button** — a circular button with the `Customer-Icon/up-top` icon. Changes appearance based on whether the user has typed text. Uses the Button/only icon pattern from `components/core/actions/button-icon`.

## Usage guidelines

**Use in chat screens only.** The Chat Input is designed for real-time messaging between two parties. Do not use it for search, form inputs, or multi-line content.

**Personalise the placeholder.** When the recipient is known, use "Message [Name]..." as the placeholder to make the context clear. Avoid generic placeholder text.

**Do not modify the wrapper padding.** The bottom padding (16px) is intentionally larger than the top — this accounts for device safe areas and thumb reach zones on mobile. Do not reduce it.

**The send button is always visible.** It is not hidden or removed when the input is empty. In the Default state it uses a passive ghost style; in the Typing state it becomes a primary dark button. This pattern communicates sendability without hiding the affordance.

**Single-line input.** The Chat Input is not a multi-line component. For notes or longer text, use Text Area.

## Do / Don't

**Do**
- Use in one-to-one chat screens where a user sends messages to a specific recipient.
- Show the recipient's name in the placeholder text when it is known.
- Keep the full wrapper padding, including the 16px bottom value.

**Don't**
- Don't use for search inputs — use a dedicated search component.
- Don't use for multi-line text — use Text Area.
- Don't remove or resize the send button.
- Don't change the border-radius of the input pill or the send button.
