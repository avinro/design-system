# Segmented Control

## Overview

The Segmented Control is a horizontal tab-strip that lets the user switch between a fixed set of mutually exclusive views or filters. Exactly one item is active at all times. It is suited for content-switching — not form submission, filtering with multiple selections, or binary toggles.

Use it when you have between two and four parallel options that need to be presented at the same visual weight on a single line.

## Anatomy

The Segmented Control is composed of two elements:

- **Track** — the outer rounded container. Carries the background color, border-radius (120px / fully rounded), inner padding (2px), and the gap between items.
- **Item** — an individual tab within the track. Each item stretches equally to fill the available track width.

## States

The Segmented Control itself has no interactive states (hover, focus, disabled) defined at the track level. State is carried by individual items.

| Item state | Description |
|---|---|
| Active | The currently selected item. Always exactly one item is active. |
| Inactive | Any unselected item. |

## Usage guidelines

**Use for content switching, not actions.** The Segmented Control switches which content is visible. Do not use it to trigger operations or submit data.

**Keep item count between 2 and 4.** Two items is the minimum; four is the maximum. If you need more options, consider a tab bar or a navigation pattern.

**Labels must be short and parallel.** All labels should be the same grammatical form (all nouns or all verbs). Labels cannot wrap — keep them to one or two words.

**One item is always active.** There is no empty or unselected state. Set a default selection before the user interacts.

**Track width.** In DS Core the track is 328px wide. In product contexts it may span the full container width. Align with the product team before deviating from the DS Core fixed width.

**Do not use as a form input.** For binary yes/no choices use Switch Toggle or Select Boolean. For selecting from a longer list use Dropdown Input or Radio Select.

## Do / Don't

**Do**
- Use for switching between 2–4 parallel content views on the same screen.
- Keep all labels concise and consistently worded.
- Always show exactly one active item.

**Don't**
- Don't use more than four items.
- Don't allow labels to wrap to a second line.
- Don't use as a substitute for Radio Select in a form.
- Don't change the number of items dynamically after the component has mounted.
