# Text Input

## Overview

Text Input is a single-line form field for free-form text entry. It is composed of three independently reusable sub-components: `input/label`, `input/field`, and `input/feedback-text`. The wrapper (`TextInput`) combines all three with a consistent 8px vertical gap and manages the overall validation state ‚Äî default, error, or succeeded.

The `input/field` sub-component supports a wider set of interaction states (focus, typing, filled, disabled) that are handled during live user interaction. The label and feedback text can be shown or hidden independently.

## Anatomy

The component stacks three elements vertically:

1. **Label** (`input/label`) *(optional)* ‚Äî composed of:
   - **Label text** ‚Äî the field name, Manrope Medium 14px
   - **Help icon** *(optional)* ‚Äî 20px question mark icon, right of the label text
   - **Helper link** *(optional)* ‚Äî right-aligned text link in the label row
   - **Description** *(optional)* ‚Äî a sub-label line of helper context in `--text/placeholder`
2. **Field** (`input/field`) ‚Äî the interactive text input, composed of:
   - **Left icon** *(optional)* ‚Äî 24px icon, left-aligned
   - **Text content area** ‚Äî shows placeholder, value, or cursor depending on state
   - **Right icon** *(optional)* ‚Äî 24px icon, right-aligned
   - **Inline button** *(optional)* ‚Äî a 32px pill-shaped action button inside the field, right-aligned
3. **Feedback text** (`input/feedback-text`) *(optional)* ‚Äî a single line below the field. Renders in three modes: info (neutral), error (red), or success (green).

## Variants

Text Input has no structural variants. Variation is expressed through props:

- **With / without label** ‚Äî controlled by `labelShow`
- **With / without feedback text** ‚Äî controlled by `message`
- **With left icon** ‚Äî optional icon inside the field, left side
- **With right icon** ‚Äî optional icon inside the field, right side
- **With inline button** ‚Äî a compact pill button embedded inside the field on the right

## Usage guidelines

- Use Text Input for free-form single-line text entry: names, numbers, card details, addresses.
- Always include a label when the field appears in a form. The label may be omitted only in highly constrained layouts where context makes the field purpose unambiguous.
- Use the description line in the label when the field requires additional instruction or context beyond the label name.
- Use the feedback text slot for helper guidance in the default state, and for validation messages in error or succeeded states. Do not use it for both at once.
- Use the inline button variant when a short contextual action (e.g. "Clear", "Apply", "Verify") is tied directly to the field value and should remain visible at all times.
- Use left or right icons to provide visual context (e.g. a phone icon for a phone number field), not for decoration.
- In forms with multiple fields, stack inputs vertically with consistent spacing.

## Accessibility

- The input field must have an accessible label ‚Äî either via a visible `<label>` element or `aria-label`/`aria-labelledby`.
- The label's help icon must have an accessible tooltip or `aria-label` that describes its purpose.
- When the field is in an error state, the error feedback text must be associated with the input via `aria-describedby`.
- The disabled state must communicate its status via `aria-disabled="true"` or a native `disabled` attribute.
- Placeholder text must not be used as a substitute for a label ‚Äî it disappears on input.
- The inline button must have its own accessible label independent of the input.

## Do / Don't

**Do:**
- Write label text as a short noun phrase (2‚Äì4 words): "Card number", "Emergency contact name"
- Write placeholder text as a format hint when relevant: "1234 5678 9012 3456", "MM/YY"
- Show the error feedback text immediately on failed validation ‚Äî do not wait for form submission

**Don't:**
- Use placeholder text as the only label ‚Äî always pair it with a visible label in forms
- Use the description slot for error messages ‚Äî use the feedback text in error state instead
- Stack multiple icons (left + right) and an inline button in the same field simultaneously
