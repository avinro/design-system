# Calendar Range

## Overview

Calendar Range is a monthly calendar that displays a single month at a time in a card surface. The user selects a continuous date range by tapping a start date and then an end date. Days between the two endpoints are highlighted with a semi-transparent band; the start and end dates receive a solid dark circle with inverted text. A today indicator and planned activity dots are also supported. The component is a composite built from two sub-components: **CalendarTitle** (navigation header, shared with Calendar Simple) and **DayItemRangeCalendar** (the only day cell, used across all states).

## Anatomy

Calendar Range is composed of the following parts:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | Outer card surface. 374px wide, border-radius 16px, subtle shadow. Background: `--surface/neutral/default`. |
| **CalendarTitle** | Yes | Navigation header: left chevron (24px) + month/year label + right chevron (24px). Shared with Calendar Simple. |
| **Weekday header** | Yes | Row of 7 abbreviated labels (SUN–SAT), equally distributed across the full width. |
| **Day grid** | Yes | 5–6 rows of 7 `DayItemRangeCalendar` cells representing the displayed month plus overflow days from adjacent months. |

### CalendarTitle sub-component

Identical to CalendarTitle in Calendar Simple. See [Calendar Simple](/components/calendar-simple) for full details.

| Part | Description |
|---|---|
| Left chevron | 24×24px icon. Navigates to the previous month. |
| Month/year label | e.g. "July 2023". Manrope Bold, `--size/base` (14px), `--text/body`. Text style: `Body sm/140/Bold`. |
| Right chevron | 24×24px icon. Navigates to the next month. |

### DayItemRangeCalendar sub-component

Each cell uses `flex: 1 0 0` width — cells fill the row equally. Height is 40px. Appearance is determined by a combination of `state`, `type`, and the `today` boolean:

| State | Type | today | Appearance |
|---|---|---|---|
| `Available` | `No-plan` | `false` | Plain date number. Color: `--day-item/text-default`. |
| `Disable` | `No-plan` | `false` | Greyed date number. Color: `--day-item/text-disabled`. Not selectable. |
| `Available` | `Planned` | `false` | Date number + 6×6px dot indicator below. Color: `--day-item/text-default`. |
| `Available` | `No-plan` | `true` | Today indicator: 36×36px circle ring outline centered in cell. Color: `--day-item/text-default`. |
| `Selected - Start` | `No-plan` | `false` | Dark 36px circle + right half-band (center → right edge). Text: `--day-item/text-selected` (inverted). |
| `Selected - Middle` | `No-plan` | `false` | Full-width semi-transparent band only — no circle. Text: `--day-item/text-selected-middle` (dark, **not** inverted). |
| `Selected - End` | `No-plan` | `false` | Dark 36px circle + left half-band (left edge → center). Text: `--day-item/text-selected` (inverted). |

## States

Three states are defined at the composite component level.

| State | Description |
|---|---|
| `Default` | Standard view. No selection, no today indicator, no planned indicators. Uses `DayItemRangeCalendar` cells. |
| `Today` | The current date is marked with a today indicator (circle ring outline). Planned activity indicators may appear. Uses `DayItemRangeCalendar` cells. |
| `Range` | A date range is selected. Start and end dates show a dark circle; all days between them show a semi-transparent band. Uses `DayItemRangeCalendar` cells. |

> Note: The today indicator and range selection can appear simultaneously — today may fall within a selected range.

## Usage guidelines

- Use Calendar Range when the user needs to select a start and end date defining a continuous period.
- The component is confirmed for use in the Dojo Customer App for date range filtering flows.
- The component is fixed-width (374px). Use it inside a modal, bottom sheet, or a dedicated screen section.
- Out-of-month overflow days use the `Disable` state to fill the 7-column grid — they are not selectable.
- Use the `Planned` type to mark days that have associated activities.
- The today indicator (`today=true`) communicates the current calendar date independently from the selection state.
- Do not use Calendar Range for single-date selection — use Calendar Simple instead.

## Accessibility

- The month/year heading should be announced as a landmark or heading for screen readers.
- Each day cell should have an accessible label with the full date (e.g., `aria-label="July 23, 2023"`), not just the number.
- Range-selected days should communicate their role: `"July 23, 2023, range start"`, `"July 24, 2023, in selected range"`, `"July 31, 2023, range end"`.
- The today indicator should be reflected in the accessible label: `"July 9, 2023, today"`.
- `Disable` days should use `aria-disabled="true"` and not receive focus.
- The left and right chevrons should have `aria-label="Previous month"` and `aria-label="Next month"`.

## Do / Don't

**Do:**
- Use for date range selection where the user needs a start and end date.
- Clearly differentiate the start, middle, and end states visually using circle + band + circle.
- Show the today indicator to help the user orient within the month.
- Use `Planned` indicators to surface relevant activity information.
- Use `--day-item/text-selected-middle` (dark) for middle-of-range day numbers — they sit on a light band, not a dark circle.

**Don't:**
- Don't use for single-date selection — use Calendar Simple instead.
- Don't make `Disable` days selectable.
- Don't omit the band connecting start and end — the visual continuity is critical for communicating the range.
- Don't apply inverted text (`--day-item/text-selected`) to middle-of-range days. Middle days use `--day-item/text-selected-middle` (#1d1d23), which is dark.
