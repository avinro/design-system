# Month Year Picker

## Overview

Month Year Picker is a drum-style (slot-machine/iOS-style) scroll picker for selecting a month and year. It displays two vertical columns — Month (right-aligned, full names) and Year (left-aligned, 4-digit) — using the same scroll depth visual system as [Time Date Picker](/components/time-date-picker).

It is used exclusively inside the **"Month and Year" bottom-sheet popup** on the Plan page. The popup is triggered by tapping the month/year label in the Plan page Header (e.g., "Jun, 2026 ▾"). Confirming a selection navigates [Calendar Plan Page](/components/calendar-plan-page) to the chosen period.

## Anatomy

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | 375×212px surface. Background: `--surface/neutral/default`. No border-radius. |
| **Selection indicator** | Yes | 31px-tall horizontal stripe centred vertically, full width. Marks the active selection row. Decorative — `aria-hidden`. |
| **Column group** | Yes | Flex row of two columns. Horizontally and vertically centred. Gap: `27px`. |
| **Month column** | Yes | Right-aligned. Full month names (January … December). 7 items always visible. |
| **Year column** | Yes | Left-aligned. 4-digit years. In `type=end`, positions 5–7 use `opacity-0` to mask future years. |

### Scroll depth visual system

Identical to [Time Date Picker](/components/time-date-picker). Each column shows 7 items; the selected value is always at position 4 (centre).

| Position | Role | Font size | Opacity | Text token |
|---|---|---|---|---|
| 1 — top | Furthest above | 12px | 40% | `--text/placeholder` |
| 2 | Near above | 14px | 70% | `--text/placeholder` |
| 3 | Close above | 18px | 100% | `--text/placeholder` |
| **4 — centre** | **Selected** | **23px** | **100%** | **`--text/primary`** |
| 5 | Close below | 18px | 100% | `--text/placeholder` |
| 6 | Near below | 14px | 70% | `--text/placeholder` |
| 7 — bottom | Furthest below | 12px | 40% | `--text/placeholder` |

> In `type=end`, positions 5–7 of the year column use `opacity: 0` instead. The month column is unaffected.

## Variants

Two variants defined by the `type` prop, representing the boundary condition of the year column:

| Variant | Prop value | Description |
|---|---|---|
| End | `end` | The most recent available year is selected. Year column positions 5–7 are `opacity-0`. |
| Middle | `middle` | A year mid-range is selected. All 7 year column positions are populated and visible. |

> Use `type=end` when the user has scrolled to the upper boundary of the available year range. Use `type=middle` for any earlier year.

## Usage guidelines

- Use Month Year Picker only inside the "Month and Year" popup. Do not use it as a standalone picker or embed it directly in a page.
- The popup is triggered by tapping the month/year label in the Plan page Header. Once a selection is confirmed, Calendar Plan Page updates to the chosen period.
- Always pair the picker with a **"Confirm Date"** primary action button. The selection must not be applied until the user confirms.
- Always include a **close button** (×) in the popup header to allow dismissal without applying changes.
- Use `type=end` when the selected year is at the upper boundary of the available range.
- Month names must be displayed in full (e.g., "January", not "Jan").

## Page composition

Confirmed structure from Dojo Customer App (Figma node `1498:25477`, "Month and Year" popup):

| Layer | Component | Responsibility |
|---|---|---|
| 1 | Overlay | Semi-transparent backdrop (`--surface/neutral/bg-overlay`) covering the Plan page |
| 2 | Popup container | Bottom-sheet panel, rounded top corners |
| 3 | Popup Header | Close (×) button · "Month and Year" title |
| 4 | **Month Year Picker** | Month and year selection |
| 5 | Action area | "Confirm Date" primary button — applies the selection |

> The popup overlays the full Plan page. The underlying Calendar Plan Page and bookings list remain visible beneath the overlay.

## Accessibility

- Each column must have a scrollable region with `role="listbox"` and `role="option"` per item.
- Provide column labels: `aria-label="Month"` and `aria-label="Year"`.
- The selected item in each column must carry `aria-selected="true"`.
- The selection indicator is decorative — mark it `aria-hidden="true"`.
- The popup must have `role="dialog"` and `aria-labelledby` pointing to the "Month and Year" heading.
- The close button must have `aria-label="Close"` or equivalent.

## Do / Don't

**Do:**
- Always show both Month and Year columns together.
- Use `type=end` when the selected year is the upper boundary of the scroll range.
- Require explicit confirmation via the "Confirm Date" button before applying the selection.
- Provide a close control to dismiss without applying changes.

**Don't:**
- Don't use this component outside of the Month and Year popup.
- Don't display a single column (month-only or year-only).
- Don't apply the selection automatically on scroll stop.
- Don't show years beyond the defined upper boundary.
