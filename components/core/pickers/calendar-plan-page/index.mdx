# Calendar Plan Page

## Overview

Calendar Plan Page is a monthly calendar component used on the Plan page of Dojo apps. It is embedded within a top sheet or drawer panel and supports two display modes: a full month grid and a collapsed single-week row (`small` state). Days can be marked as planned using an orange dot indicator. Unlike [Calendar Simple](/components/calendar-simple), this component has no navigation controls of its own and is not a general-purpose date picker.

**Calendar Plan Page always coexists with a Header component** placed directly above it. The Header is responsible for displaying the current month and year, providing back navigation, and offering controls to change the month, year, or week. The calendar is a passive display ŌĆö it renders the period that the Header sets.

It is built from one internal sub-component:
- **DayItemSimpleCalendar** ŌĆö individual 40├Ś40px day cell, shared with [Calendar Simple](/components/calendar-simple).

## States

Four states are defined at the composite component level via the `state` prop.

| State | Description |
|---|---|
| `Default` | Full month view. No date selected. No today highlight. Planned indicators visible on applicable days. |
| `Today` | Full month view. Today's date is selected by default (dark circle). Planned indicators visible. |
| `Active` | Full month view. A non-today date is selected (dark circle). Today's date shows a lighter circle highlight. |
| `small` | Collapsed view. Only the current week row is visible. Overflow clipped to 96px height. Selected and today highlights are preserved. |

> The `small` state is used when the Plan page is in a compact or summary mode ŌĆö for example, when the user scrolls down and the calendar collapses to the current week.

## Usage guidelines

- Use Calendar Plan Page only in the Plan page context. It is not a general-purpose date picker ŌĆö use [Calendar Simple](/components/calendar-simple) for standalone date selection.
- Always render Calendar Plan Page together with a Header component placed directly above it. The Header owns all navigation: displaying the current month/year label, changing the month, week, or year, and back navigation. Do not add any navigation controls inside the calendar itself.
- The calendar responds to the period set by the Header. When the user changes the month or year via the Header, the calendar re-renders to show the selected period.
- Use the `small` state when the plan page is in a compact or summary mode, collapsing the calendar to the current week row.
- Mark days with a planned job or delivery using `type="Planned"` on the relevant `DayItemSimpleCalendar` cells.
- Out-of-month overflow days must use `state="Disable"` to maintain the 7-column grid structure. They must not be interactive.
- The handle at the bottom is always visible. Wire it to expand/collapse behaviour between `small` and full states.

## Page composition

Calendar Plan Page is always used as part of a composed page layout. Confirmed structure from Dojo Customer App (Figma node `3028:4306`):

| Layer | Component | Responsibility |
|---|---|---|
| 1 | System bar | Status bar |
| 2 | **Header** | Back navigation ┬Ę Current month/year label ┬Ę Month/year change (dropdown) ┬Ę Search ┬Ę Add action |
| 3 | **Calendar Plan Page** | Day grid display ┬Ę Day selection ┬Ę Planned indicators ┬Ę Collapse/expand |
| 4 | Content area | Bookings list or empty state |

> The Header component controls what period the calendar displays. Calendar Plan Page has no awareness of or responsibility for navigation between months, weeks, or years.

## Accessibility

- Each day cell must have an `aria-label` with the full date, e.g., `aria-label="March 9, 2026"`.
- Planned days must include an accessible cue in the label, e.g., `aria-label="March 9, 2026, has planned activity"`.
- Out-of-month cells must use `aria-disabled="true"` and should not receive keyboard focus.
- Weekday header cells are decorative ŌĆö mark them `aria-hidden="true"`.
- The handle must have an accessible label such as `aria-label="Collapse calendar"` when interactive.

## Do / Don't

**Do:**
- Use the `small` state to collapse the calendar when the user scrolls past the plan view.
- Use `Planned` to communicate that a job or delivery is scheduled on a specific day.
- Preserve the selected day and today highlight when switching between `small` and full states.
- Provide full date `aria-label` values on every day cell.

**Don't:**
- Don't use this component outside of a top sheet or drawer context.
- Don't use it as a standalone date picker ŌĆö use Calendar Simple instead.
- Don't make `Disable` days selectable or focusable.
- Don't add navigation controls inside this component ŌĆö all period navigation belongs in the Header above.
- Don't render Calendar Plan Page without its companion Header.
