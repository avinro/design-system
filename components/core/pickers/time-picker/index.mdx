# Time Picker

## Overview

Time Picker is a drum-style (slot-machine/iOS-style) scroll picker for selecting a time of day. It displays three vertically scrolling columns — Hour, Minute, and AM/PM — using the same scroll depth visual system as [Time Date Picker](/components/time-date-picker), [Month Year Picker](/components/month-year-picker), and [Number Picker](/components/number-picker).

It is used inside bottom-sheet popups when the user needs to choose an arrival or appointment time. The popup is triggered by a "Select Time" action in a Dojo AI chat conversation. Confirming the selection applies the chosen time.

> **Note:** Time Picker (3 columns: Hour, Minute, AM/PM) is distinct from [Time Date Picker](/components/time-date-picker) (4 columns: Date, Hour, Minute, AM/PM). Use Time Picker when no date column is required.

## Anatomy

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | 375×212px surface. Background: `--surface/neutral/default`. No border-radius. In-app context renders on dark surface (`--bg-surface-default`). |
| **Selection indicator** | Yes | 31px-tall horizontal stripe centred vertically, full width. Marks the active selection row. Decorative — `aria-hidden`. |
| **Column group** | Yes | Flex row of three columns. Centred horizontally (with a 4px right offset). Gap: `27px`. |
| **Hour column** | Yes | Center-aligned. 12-hour format integers (1–12). 7 items, wraps cyclically. |
| **Minute column** | Yes | Center-aligned. Zero-padded integers (00–59). 7 items, wraps cyclically. |
| **AM/PM column** | Yes | Center-aligned. Only 2 of 7 slots are populated (AM and PM). Unpopulated slots use `opacity: 0`. |

### Scroll depth visual system

Identical to [Time Date Picker](/components/time-date-picker), [Month Year Picker](/components/month-year-picker), and [Number Picker](/components/number-picker). Each column shows 7 items; the selected value is always at position 4 (centre).

| Position | Role | Font size | Opacity | Text token |
|---|---|---|---|---|
| 1 — top | Furthest above | 12px | 40% | `--text/placeholder` |
| 2 | Near above | 14px | 70% | `--text/placeholder` |
| 3 | Close above | 18px | 100% | `--text/placeholder` |
| **4 — centre** | **Selected** | **23px** | **100%** | **`--text/primary`** |
| 5 | Close below | 18px | 100% | `--text/placeholder` |
| 6 | Near below | 14px | 70% | `--text/placeholder` |
| 7 — bottom | Furthest below | 12px | 40% | `--text/placeholder` |

### AM/PM column slot layout

The AM/PM column has a fixed 2-value layout. All unpopulated positions use `opacity: 0`:

| Period selected | Position 3 | Position 4 (selected) | Position 5 |
|---|---|---|---|
| **AM** | `opacity: 0` | **AM** | PM |
| **PM** | AM | **PM** | `opacity: 0` |

Positions 1, 2, 6, and 7 are always `opacity: 0` in this column.

## Variants

Two variants defined by the `period` prop (Figma: `PM` boolean), representing which time period is selected:

| Variant | Prop value | Figma name | Description |
|---|---|---|---|
| AM | `"AM"` | `PM=off` | AM is selected (position 4). PM appears at position 5. |
| PM | `"PM"` | `PM=on` | PM is selected (position 4). AM appears at position 3. |

> Use `period="AM"` for morning times and `period="PM"` for afternoon and evening times.

## Usage guidelines

- Use Time Picker only inside a bottom-sheet popup. Do not embed it directly in a page or form.
- Always pair with a confirm action button. The selection must not be applied until the user explicitly confirms.
- Always include a **close button** (×) in the popup header to allow dismissal without applying changes.
- Use `period="AM"` for morning times and `period="PM"` for afternoon and evening times.
- Hour values are in 12-hour format (1–12) — not 24-hour.
- Minute values are zero-padded to 2 digits (e.g., "09", "41").
- When a date is also required alongside the time, use [Time Date Picker](/components/time-date-picker) instead.

Confirmed structure from Dojo Customer App (node `222:9659`, "Choose arrive time" popup):

| Layer | Component | Responsibility |
|---|---|---|
| 1 | Overlay | Semi-transparent backdrop (`rgba(0,0,0,0.58)`) covering the chat screen |

| 3 | Popup Header | Close (×) icon · "Choose arrive time" title |
| 4 | **Time Picker** | Hour, Minute, and AM/PM selection |
| 5 | Action area | "Confirm Time" primary button — applies the selection |

> The popup is triggered by the "Select Time" button in a Dojo AI chat message. It overlays the chat screen. The underlying conversation remains visible beneath the overlay.

## Accessibility

- Each column must have a scrollable region with `role="listbox"` and `role="option"` per item.
- Provide column labels: `aria-label="Hour"`, `aria-label="Minute"`, `aria-label="AM or PM"`.
- The selected item in each column must carry `aria-selected="true"`.
- The selection indicator is decorative — mark it `aria-hidden="true"`.
- The popup must have `role="dialog"` and `aria-labelledby` pointing to the "Choose arrive time" heading.
- The close button must have `aria-label="Close"` or equivalent.

## Do / Don't

**Do:**
- Always show all three columns (Hour, Minute, AM/PM) together.
- Pair with a confirm action button — do not apply the selection on scroll stop alone.
- Provide a close button to allow dismissal without changes.
- Use `period="PM"` when the initial time is in the afternoon or evening.

**Don't:**
- Don't use this component outside of a popup context.
- Don't apply the selection automatically on scroll — wait for explicit confirmation.
- Don't use a 24-hour format — the component displays 12-hour time with AM/PM.
- Don't use Time Picker when a date is also required — use [Time Date Picker](/components/time-date-picker) instead.
