# Button Vertical

## Overview

Button Vertical is a stacked action button used in contextual action grids inside bottom sheets. It displays a colored shape (diamond or circle) with an icon overlaid, and a text label below. It is used when multiple related actions need to be presented as a visual grid — for example, map reporting options or categorized contextual shortcuts in the Dojo Driver App.

Unlike other button components in this system, it is not a standalone trigger — it is always used as part of a multi-item action grid.

## Anatomy

The Button Vertical is composed of four parts:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | Rounded-square tile (~120px wide, 96px tall). Gray neutral surface background. Vertical flex layout with centered alignment. |
| **Icon-form shape** | Yes | A colored background shape (diamond or circle) that sits behind the icon. Communicates the action category through color. |
| **Icon** | Yes | A 32×32px action icon centered on and overlaid over the icon-form shape. |
| **Label** | Yes | Text label below the icon stack. Manrope Medium 16px, center-aligned. Can wrap to two lines. |

## Variants

The configurable property is **Icon-Form** — the shape of the background element behind the icon.

### Icon-Form: diamond

A rotated square (diamond) shape. 54×54px. The diamond is color-coded to communicate the category of the action.

Use for actions that belong to a category identifiable by color — for example, incident types in a map report grid.

### Icon-Form: circle

A circular ellipse shape. 48×48px. Slightly smaller than the diamond form.

Use for actions that do not require category color-coding or that belong to a more neutral, uncategorized context.

## States

Button Vertical supports four states. There is no hover state — this component is designed for touch and mobile contexts.

| State | Trigger |
|---|---|
| `active` | Resting state. The button is available and not interacted with. |
| `pressed` | Button is actively tapped or clicked. |
| `disable` | Action is unavailable. Non-interactive. |
| `focus` | Button receives keyboard focus. |

> Note: The resting state is named `active` in Figma (not `default`).

## Usage guidelines

- Use Button Vertical **only in action grids inside bottom sheets**. Do not use it as a standalone button in regular layouts.
- Arrange instances in a **2-column grid** as the default layout. TODO: Confirm grid column count and item spacing.
- Use the `diamond` form for actions that are color-coded by category (e.g., incident type, hazard level).
- Use the `circle` form for neutral or uncategorized actions.
- Use consistent icon-form types within the same grid. Avoid mixing diamond and circle forms in the same action sheet without a clear reason.
- Keep labels concise. One word is preferred. Two short words are acceptable. Avoid labels that require three or more lines.
- Do not use this component for primary navigation or persistent UI actions.

## Accessibility

- The component has a visible label. `aria-label` is not required unless the label alone is ambiguous in context.
- The icon-form shape and icon should be `aria-hidden="true"` — the label carries the accessible name of the button.
- The `focus` state is designed and present. Do not suppress the focus ring.
- Implement `disabled` or `aria-disabled="true"` for disabled instances. Visual state alone is not sufficient.

**Do:**
- Use in action grids inside bottom sheets.
- Arrange in a 2-column grid.
- Use the diamond form for color-coded category actions.
- Keep labels short — one word preferred, two if necessary.
- Use consistent icon-form types within the same grid.

**Don't:**
- Don't use as a standalone button outside of a grid context.
- Don't mix diamond and circle forms in the same action grid without a clear reason.
- Don't write labels that require more than two lines.
- Don't use this component for primary navigation or persistent UI controls.
