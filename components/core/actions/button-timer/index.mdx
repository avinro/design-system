# Button Timer

## Overview

Button Timer is a pill-shaped dark button that communicates urgency through a countdown visualization. A dark fill overlay grows from left to right over the button surface as time expires, progressively darkening it to signal that the available window is closing.

In the Dojo Driver App, it is used in the **"Finding a trip → New trip flow"** — shown when a new trip is offered to the driver -, and **"Adding Accident report to the map"** — shown when adding a map report - with a countdown to accept. As the countdown progresses, the button darkens, reinforcing the need to act quickly.

## Anatomy

The Button Timer is composed of three parts:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | Dark pill-shaped button (303×56px, border-radius 120px). Background: `--button/default/bg-default` (`#1d1d23`). |
| **Fill overlay** | Yes | Absolutely positioned dark layer that grows from left to right. Uses `mix-blend-multiply` to progressively darken the button as the countdown progresses. |
| **Label** | Yes | Text centered over the button. Uses `mix-blend-difference` to remain readable over both unfilled and filled areas at all animation positions. |

- All three parts are required. The fill overlay is integral to the countdown visualization — do not omit it.
- Do not replace the fill overlay with a border, progress bar, or ring indicator. The full-surface fill is the confirmed visual mechanic.

## States

Three visual states are defined in Figma, representing snapshots of the countdown fill animation.

| State | Fill overlay width | Description |
|---|---|---|
| `full` | 0px | Timer just started. No fill overlay visible. Full time remaining. |
| `middle` | ~147px (~48.5%) | Timer approximately halfway. Left half of button is darkened. |
| `state3` | 303px (full width) | Timer nearly expired. Entire button is darkened. |

> Note: These states are animation snapshots, not discrete interactive states. The actual fill grows continuously from `full` to `state3`. No hover, pressed, disabled, or focus states are defined in Figma.

## Usage guidelines

- Use Button Timer **only for timed action prompts** where the user must act within a limited countdown window. The fill animation must correspond to real elapsed time.
- Use a **single Button Timer per screen**. It represents the primary timed action of that context.
- Do not use Button Timer for untimed confirmations — use Button or Button Slider instead.
- Write a short, clear label that names the action: "Accept", "Confirm", "Start". Avoid vague labels.
- The component has a fixed width of 303px. Do not stretch it to fill the screen width.
- Drive the fill overlay width in real-time from the countdown timer — do not animate it independently.

## Interaction

The Button Timer is a timed prompt component, not a repeating interactive control.

- The fill overlay grows continuously from 0px to 303px as the countdown progresses.

## Do / Don't

**Do:**
- Use for a single, timed acceptance prompt per screen.
- Write a short, action-specific label ("Accept", "Confirm").
- Drive the fill overlay width from a real countdown timer.
- Preserve `mix-blend-multiply` on the fill overlay and `mix-blend-difference` on the label in all implementations.

**Don't:**
- Don't use without an actual countdown — the fill must correspond to real elapsed time.
- Don't place more than one Button Timer on a screen.
- Don't reverse the fill direction (left to right is the confirmed direction).
- Don't replace the fill overlay with a separate progress bar or ring indicator.
- Don't use for actions without a time limit.
