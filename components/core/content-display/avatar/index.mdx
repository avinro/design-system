# Avatar

## Overview

Avatar is a circular display component that shows a user's profile photo. When no photo is available — or while one is loading — it falls back to a neutral silhouette placeholder. Avatar is a display-only component with no interactive states.

It is used throughout the Dojo Apps to represent users and drivers in context: profile screens, trip summaries, driver cards, ratings, and chat headers.

## Anatomy

The Avatar is composed of up to two parts:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | Fully circular frame sized to the chosen size. Clips the content (photo or placeholder) to a circle. |
| **Photo** | Conditional | User's profile photo, clipped to fill the container (`object-cover`). Shown when a valid image is available. |
| **Placeholder** | Conditional | Silhouette icon on a neutral background. Shown when no photo is provided or when the photo fails to load. |

- Photo and placeholder are mutually exclusive — exactly one is visible at all times.
- The container always has `border-radius: 50%`. The shape is never square or rounded-square.
- The photo always fills the container without letterboxing — do not add internal padding.

## States

Avatar has no interactive states. It is a display-only component.

| Content state | Description |
|---|---|
| With photo | A valid user photo is available and displayed, clipped to a circle. |
| Placeholder | No photo is available. A silhouette icon on a neutral background is shown. |

## Usage guidelines

- Use Avatar to represent a **specific person**: a user, a driver, or a customer.
- Choose the **size** based on the visual hierarchy of the screen:
  - `Largest` (96px) or `XXL` (72px) — primary profile displays or featured user headers.
  - `XL` (56px) or `LG` (48px) — trip summaries, driver cards, and secondary user references.
  - `MD` (40px) or `SM` (32px) — list rows, chat items, and compact contexts.
  - `XS` (24px) — inline or dense UI where only a small visual identifier is needed.
- Always show the **placeholder silhouette** when no photo is available. Never leave the avatar empty.
- Do not use Avatar to represent non-person entities (e.g., brands or products) without a confirmed design decision.
- Do not display text initials inside the avatar unless an initials variant is confirmed.

## Accessibility

- Avatar is a display element. Provide a meaningful `alt` attribute on the photo image equal to the person's name (e.g., `alt="Maria García"`).
- When the avatar is purely decorative (e.g., the name is visible in an adjacent label), use `alt=""` on the image to mark it as decorative.
- Do not rely on the avatar alone to communicate identity — always pair it with a visible name label where the layout allows.
- Avatar is not focusable and should not receive keyboard focus unless it is interactive in a specific context (e.g., tapping opens a profile screen). If made interactive, apply appropriate `role` and focus treatment.

## Do / Don't

**Do:**
- Use to represent a specific person (user, driver, customer).
- Always show the placeholder silhouette when no photo is available.
- Choose the size appropriate to the screen's visual hierarchy.
- Provide a meaningful `alt` attribute when the image represents a named person.

**Don't:**
- Don't display an empty circle or a broken image — always fall back to the placeholder.
- Don't change the shape — Avatar is always fully circular.
- Don't display text initials unless an initials variant is confirmed.
- Don't use Avatar to represent non-person entities without a confirmed design decision.
- Don't add decorative borders or rings unless explicitly required by a specific context.
