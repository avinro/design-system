# Tabs

## Overview

Tabs is a full-width underline tab bar used to switch between content sections on the same screen. Each tab occupies an equal share of the bar width. The active tab is indicated by a 2px dark bottom border. Exactly one tab is active at all times.

Use Tabs when a screen contains multiple parallel content views that the user should be able to switch between without navigating away.

## Anatomy

The Tabs component has three parts:

- **Tab bar** — the outer container. Full width, with horizontal padding on both sides and a 1px bottom divider line. No background in DS Core.
- **Tab item (active)** — the currently selected tab. A 2px dark bottom border serves as the active indicator. Text or icon renders in the active dark color.
- **Tab item (inactive)** — any unselected tab. No visible bottom border. Text or icon renders in a muted color.

The base component for each item is `tab-items` (`5001:2072`), which has four configurations: text-only active, text-only inactive, icon-only active, icon-only inactive.

### Token reference

#### Tab bar

| Token | Value | Description |
|---|---|---|
| `--notification/border-default` | `#e4e5ed` | Bottom divider line color. DS Core and Customer App use this token name. Driver App uses `--border-default` for the same value. |
| border-bottom-width | `1px` | Divider line width |
| `--spacing-12` | `12px` | Horizontal padding |
| height | `44px` | Tab bar height |
| background | none (DS Core) | DS Core has no background. Driver App uses `--bg-surface-default` / `#f1f2f4`. Confirm before implementing. |

#### Active tab item

| Token | Value | Description |
|---|---|---|
| `--tab/border-active` | `#1d1d23` | Active indicator bottom border color |
| `--sds-size-stroke-focus-ring` | `2px` | Active indicator border-bottom-width |
| `--tab/text-active` | `#1d1d23` | Active tab text / icon color |

#### Inactive tab item

| Token | Value | Description |
|---|---|---|
| `--tab/bg` | `rgba(29, 29, 35, 0)` | Inactive tab background (transparent) |
| `--tab/border-inactive` | `rgba(29, 29, 35, 0)` | Inactive tab bottom border (transparent) |
| `--tab/text-inactive` | `#636377` | Inactive tab text / icon color (DS Core / Customer App). Driver App uses `#9192af`. Confirm intended value. |
| border-bottom-width | `1px` | Inactive tab border-bottom (transparent, maintains layout alignment) |

**Note on token naming in the Driver App:** The Driver App uses hyphen-notation tokens (`--tab-border-active`, `--tab-text-active`) and contains a typo for "inactive" (`--tab-border-inative`, `--tab-text-inative` — missing "c"). Use DS Core slash-notation token names as the source of truth.

## Usage guidelines

**Use for content-switching, not navigation.** Tabs switch between sections of content on the same screen. They are not a substitute for a bottom navigation bar or an app-level navigation pattern.

**Keep item count between 2 and 4.** Fewer than 2 tabs offers no choice; more than 4 creates a crowded bar where labels may be truncated.

**Labels must be short and parallel.** One-word labels are strongly preferred. All labels should follow the same grammatical form. Labels cannot wrap.

**One tab is always active.** There is no unselected state. Set a default active tab before the user interacts.

**Equal-width tabs.** Each tab item fills `flex: 1` of the bar. Do not set explicit widths per item.

**Tab bar background.** The DS Core definition has no background on the tab bar. The Driver App adds a surface background. Align with the product team on the intended behavior for your context before deviating from DS Core.

**Paired with content.** Tabs do not render content themselves — they signal which content panel should be visible. The content panel must be managed by the parent screen.

## Do / Don't

**Do**
- Use for 2–4 parallel content sections within the same screen.
- Keep labels short, centered, and consistent.
- Always show exactly one active tab.

**Don't**
- Don't use more than four tabs.
- Don't use as a top-level app navigation bar.
- Don't allow tab labels to wrap to a second line.
- Don't set different widths for individual tab items.
- Don't hide or remove tabs dynamically based on user state.
