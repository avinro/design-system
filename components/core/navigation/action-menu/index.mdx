# Action Menu

## Overview

Action Menu is a list of contextual actions presented in a contained surface. Each item consists of an icon and a label. The component supports two presentation styles: a **Sheet** that slides up from the screen edge, and a **Popover** that floats near a trigger element.

It is used in the Dojo Apps to surface contextual options relevant to the current screen or selected element.

## Anatomy

The Action Menu is composed of the following parts:

| Part | Required | Description |
|---|---|---|
| **Container** | Yes | The outer surface holding the menu items. Background: `--surface/neutral/default` (`#f1f2f4`). Shape and shadow differ by variant. |
| **Handle** | Sheet only | A 32×4px drag handle pill at the top of the Sheet variant. Indicates the sheet can be dismissed by dragging. Background: `--handler/bg` (`#e4e5ed`). |
| **Menu item** | Yes | A single action row (44px height). Contains an icon (20×20px) and a label. |

### Menu item anatomy

Each item shares the same layout across both variants:

| Property | Value |
|---|---|
| Height | 44px |
| Horizontal padding | 16px |
| Vertical padding | 5px |
| Icon size | 20×20px |
| Gap (icon → label) | 16px |

- Every item requires both an icon and a label. Neither can be omitted.
- Icons are 20×20px. Use icons from the confirmed `Customer-Icon` set.

## Variants

Action Menu has two variants, corresponding to different presentation contexts.

### Sheet

A full-width action surface with a drag handle at the top. Intended for bottom sheet presentation — it slides up from the bottom of the screen.

| Property | Value |
|---|---|
| Width | 340px |
| Border radius | 12px |
| Shadow | None |
| Handle | Yes — 32×4px pill, `--handler/bg` |

Use for primary contextual action sets triggered from a bottom action area.

### Popover

A compact floating menu with a drop shadow. Anchored near the element that triggered it — typically a "more options" icon or overflow button.

| Property | Value |
|---|---|
| Width | 180px |
| Border radius | 8px |
| Shadow | `elevation/dark/02` — `0px 8px 12px rgba(0,0,0,0.16)` |
| Handle | None |

Use for secondary or overflow actions triggered inline.

> Note: The Figma variant name for Sheet is "Variant3". The Figma variant name for Popover is "Popover-view".

|---|---|

## Usage guidelines

- Use Action Menu to surface a **focused set of contextual actions** — only actions directly relevant to the current state or selected element.
- Choose the **Sheet** variant when the menu is triggered from a bottom action area or when the actions are the primary decision point on the screen.
- Choose the **Popover** variant when the menu is triggered inline — for example, from a "more options" or overflow icon.
- Use **3 items** for tightly scoped contexts. Use **6 items** when more options are genuinely needed — prefer fewer items where possible.
- Write **short, action-oriented labels**: one verb + optional object (e.g., "Cancel ride", "Share location", "Report issue").
- Do not use Action Menu for navigation or non-action items. It is for actions only.
- Do not use Action Menu as a primary navigation surface.

## Accessibility

- The menu container should use `role="menu"` and each item `role="menuitem"`.
- Focus must move into the menu when it opens and return to the trigger element when it closes.
- Each item's accessible label must match its visible text.
- The popover trigger should use `aria-haspopup="menu"` and `aria-expanded` to communicate menu state.

## Do / Don't

**Do:**
- Use for a focused set of contextual actions (3–6 items).
- Write short, action-oriented labels (verb + object).
- Choose Sheet for bottom-triggered menus; Popover for inline overflow menus.
- Include both an icon and a label on every item.

**Don't:**
- Don't surface more than 6 actions — reconsider the information architecture.
- Don't use for navigation or non-action items.
- Don't use the Sheet variant as an inline element; don't use the Popover as a full-screen overlay.
- Don't omit the icon or label from any menu item.
- Don't write vague labels like "More" or "Options" — always describe the specific action.
