## Overview

The color system is organized in two layers: a **raw palette** of named color values, and a **semantic layer** that maps those values to interface roles. Components always reference semantic tokens — never raw palette values directly.

---

## Semantic tokens

Semantic tokens describe the role of a color, not its value. This decouples components from specific hues, making it possible to adjust the palette or support theming without touching component code.

### Foreground

Used for text, icons, and interactive elements rendered on top of a background.

| Token | Value | Use |
|---|---|---|
| `foreground.neutral.strongest` | `#1d1d23` | Primary text, headings |
| `foreground.neutral.strong` | `#34353f` | Body text |
| `foreground.neutral.medium` | `#4b4c5b` | Secondary text |
| `foreground.neutral.subtle` | `#636377` | Placeholder, caption |
| `foreground.neutral.lowest` | `#7a7b93` | Disabled text |
| `foreground.neutral.inverse` | `#f1f2f4` | Text on dark backgrounds |

## Raw palette

Raw palette entries are the underlying, non-semantic values. Use them only to define semantic tokens — never inside components.

| Group | Steps available |
|---|---|
| `palette.neutral` | white, black, 100, 300, 600, 900 |
| `palette.red` | 700 |
| `palette.blue` | 500, 700 |
| `palette.green` | 500, 700 |
| `palette.yellow` | 400 |
| `palette.static` | black (`#020202`), white (`#ffffff`) |

<Note>
The palette is intentionally sparse. Color ramps are only partially defined in the current Figma file. Do not extend components to use unlisted palette steps without first adding them to the token file.
</Note>

---

## Usage principles

- Always reference a semantic token. Do not hardcode hex values in components.
- Use `foreground.neutral.*` for text and icons on light backgrounds. Use `foreground.neutral.inverse` on dark backgrounds.
- Apply `surface` tokens based on z-level. `bg-sunken` sits below the page default, `bg-raised` sits above it, `bg-overlay` covers everything.
- Use `notification.*` tokens only for status feedback — error, success, warning, and info. Do not repurpose them for decorative color.
- Do not reference `palette.*` tokens in components or design specs.
